## 4. 一套引擎，所有"管理"都通用

```
                    ASTRAION 引擎
                          │
         ┌──────┬────────┼────────┬──────┐
         │      │        │        │      │
         ▼      ▼        ▼        ▼      ▼
      用户  角色  权限  流程  文档  客户  订单  ...
     管理  管理  管理  管理  管理  管理  管理

     对引擎来说：都是"一个模型 + 若干字段 + 通用CRUD"

     用户 = { model: "user", fields: [username, password, email...] }
     客户 = { model: "customer", fields: [name, phone, email...] }

     引擎不区分内置还是业务，一视同仁
```

## 11. AI Tool 注册流程

```
1. ASTRAION 启动
2. 初始化元数据表（首次/已有）
3. 读取所有模型定义（内置 + 业务）
4. 为每个模型自动生成 CRUD Tool
       ↓
   Tool: create_{modelName}
   Tool: query_{modelName}
   Tool: update_{modelName}
   Tool: delete_{modelName}
       ↓
5. 注册系统 Tool
       ↓
   Tool: createModel / updateModel / deleteModel
   Tool: createUser / setPermission
   Tool: createWorkflow / submitWorkflow / approveTask
   Tool: addDatasource / importTable
   Tool: createAgent / activateAgent (v2)
       ↓
6. 注册到 ToolRegistry
       ↓
7. AI Orchestrator 加载 System Prompt：
   "你可以使用以下工具：..."
       ↓
8. 用户建新模型 → 自动重复 3-4 步
   新 Tool 立即可用
```

### Tool 权限过滤

```
AI Orchestrator 根据当前用户 role 过滤 Tool：

root → 所有 Tool 可见
admin → 系统初始化 Tool 不可见（api_config, db_config）
user → 仅业务操作 Tool 可见

管理员看不到 API Key 相关的 Tool
普通用户看不到模型管理和用户管理的 Tool
```
