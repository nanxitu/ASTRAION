## 1. 核心理念

**ASTRAION** = ASTRA（星辰） + ION（造物粒子）  
**星汉造物 · AI 管理一切**

ASTRAION 是一套 **元数据驱动的通用业务运行时架构**，颠覆传统的 BS/CS 七层架构（Entity → Repository → Service → Controller → Vue Page）。

### 核心主张

```
传统软件开发的病：
  加一个字段 → 改 7 层代码 → 部署 → 重启
  加一个业务 → 写整个模块 → 从数据库到前端画页面

ASTRAION 的解法：
  管理员说一句话 → AI 理解 → 配置元数据 → 即时生效
  不需要写代码、不需要编译、不需要部署、不需要重启
```

### All in AI

**ASTRAION 是彻底 AI 原生的系统。** 除了登录页面，一切交互都通过对话完成：

- ✅ 系统初始化 → 对话完成
- ✅ 配置 AI 模型 → 对话完成
- ✅ 创建业务模型 → 对话完成
- ✅ 管理用户权限 → 对话完成
- ✅ 操作业务数据 → 对话完成

**不设任何传统配置界面。只有一个登录页 + 一个聊天页。**

```
  ┌─────┼─────┐

数据库  AI模型  管理员
配置    配置    创建
  │     │     │
  └─────┼─────┘
        │
        ▼
┌──────────────────────────────┐
│  ✅ 初始化完成                 │

│  管理员 tunx 登录接管整个系统    │

```

### 初始化详情

**第 1 步：数据库配置**
- 数据库类型（PostgreSQL 推荐）
- 连接地址、端口、库名
- 用户名、密码
- 连接池参数
- 测试连接 → 初始化元数据表

**第 2 步：AI 模型配置**
- 选择提供商（DeepSeek / OpenAI / 通义千问 / Ollama 本地）
- 输入 API Key
- 配置 Base URL
- 模型参数（temperature / max_tokens）
- 测试连接 → AI 激活

**第 3 步：创建管理员**
- 输入管理员用户名和密码
- 管理员创建成功
- 退出 root，管理员接管

## 5. 核心概念

### 5.0 演示数据（OA 场景）

系统内置一键加载的 OA 演示数据：7 个模型（部门/员工/请假/报销/公告/会议室/会议预约），30+ 条示例数据。Admin 可随时清除，退出演示模式。

### 5.1 元数据模型 (ModelMeta)
一切业务的起点。管理员通过对话定义模型，ASTRAION 自动完成所有后续工作。

**一个模型 = 一个数据库表 + 一套 CRUD API + AI 自动渲染**

### 5.2 内置模型 vs 业务模型
| 类型 | 说明 | 示例 |
|------|------|------|
| 内置模型 | 系统预置，不可删除 | user, role, permission, workflow, document, agent |
| 业务模型 | 管理员通过对话创建 | customer, order, product, leave_request |

### 5.3 通用运行时引擎
引擎不区分"这是用户管理"还是"这是客户管理"——它只知道有一个模型 X、有这些字段、按字段建表、生成 CRUD、校验权限。

### 5.4 插件机制
复杂业务逻辑通过插件实现（独立 jar 包，热加载，不重启）。

### 5.5 智能体系统（v2 预留）
未来支持多个 AI Agent 各司其职——销售助手、审批助手、报表助手等，每个 Agent 独立配置模型、提示词和可用工具，由 Agent 调度器自动路由。

## 6. 设计原则

1. **All in AI** — 登录后，一切操作都是对话，不设传统配置页面
2. **对话即配置** — 一切业务通过自然语言定义
3. **元数据驱动** — 数据和行为由元数据描述，引擎解析执行
4. **通用至上** — 用户管理 = 客户管理，对引擎无区别
5. **零代码扩展** — 复杂逻辑用插件，热加载不重启
6. **旧系统兼容** — 支持反向工程接入现有数据库
7. **渐进披露** — AI 按需展示数据，不堆砌信息
8. **绝对安全** — API Key 和数据库配置仅 root 可见，管理员都看不到
9. **面向未来** — 为智能体系统预留完整扩展点
