## 二、配置安全策略

| 存储位置 | 内容 | 原因 |
|---------|------|------|
| **application.yml** | 数据库连接参数 | 系统启动必需 |
| **数据库 astraion_system_config** | AI Key、AI 配置、系统标记 | 防止泄露到源码/配置文件 |

### AI Key 保护
- ✅ 初始化时 API Key 仅存数据库
- ✅ 读取时脱敏返回（前4 + 后4）
- ✅ application.yml 中绝无 AI Key
- ✅ Admin 可修改 AI 配置但看不到完整 Key

## 三、演示数据

### OA 演示模型

系统内置一键加载的 OA 演示数据，用于展示 ASTRAION 能力：

| 模型 | 字段 | 示例数据 |
|------|------|---------|
| department（部门） | name, description, manager_name | 4 个部门 |
| employee（员工） | name, dept_id, position, email, phone | 10 名员工 |
| leave_request（请假） | employee_id, leave_type, start_date, end_date, status | 5 条（年假/事假/病假/婚假） |
| expense_report（报销） | employee_id, expense_type, amount, status | 5 条（差旅/招待/办公/培训） |
| announcement（公告） | title, content, level, publisher | 3 条 |
| meeting_room（会议室） | code, type, capacity, location | 4 间 |
| meeting_booking（会议） | room_id, employee_id, title, date | 4 条预约 |

### 操作方式

- **加载**：admin 登录 → 侧栏「📦 加载 OA 演示数据」→ 自动创建 7 个模型 + 30+ 条数据
- **清除**：侧栏「🗑 清除演示数据」→ 删除所有演示模型及数据，系统恢复干净
- **权限**：仅 admin 可操作；root 不可创建业务模型

### 演示场景

加载后可对话测试典型 OA 操作：
- 「显示所有部门」
- 「张伟是谁」（关联查询）
- 「有哪些待审批的请假」
- 「帮我建一个员工叫李明」
- 「今天有哪些会议」

java -jar astraion-server-1.0.0-SNAPSHOT.jar
